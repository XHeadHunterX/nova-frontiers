Nova Frontiers floating chat/messages patch

Changed files after running updater:
- backend/app/main.py
- frontend/src/main.jsx
- frontend/src/styles.css

Backend additions:
- Adds /api/chat/messages GET endpoint.
- Adds /api/chat/messages POST endpoint.
- Adds /api/messages/threads GET endpoint.
- Adds /api/messages/thread GET endpoint.
- Adds /api/messages/send POST endpoint.
- Creates chat_messages, chat_rate_limits, and direct_messages tables if missing.
- Chat rate limits:
  - 1 message per player every 3 seconds.
  - 5 messages per player every 20 seconds.
  - 60 second cooldown when the burst cap is exceeded.
- Chat retention:
  - Keeps only the latest 100 messages per chat scope.
  - Scope means global, faction, or guild.

Frontend additions:
- Hides/removes Chat from visible nav controls by DOM scan.
- Adds a minimized floating chat icon at bottom right on every page.
- Expands into an 80% opacity chat panel over the current page.
- Fetches messages only when opened.
- Polls/streams only while opened.
- Supports Global, Faction, and Guild tabs.
- Adds direct-message overlay available at #messages or /messages.
- Adds Message buttons to profile-like DOM nodes that expose data-user-id or data-player-id.

Notes:
- This patch is intentionally isolated. It does not modify the existing build_state hot path.
- The floating chat reads /api/state only when opened to identify the current player/faction/guild.
- Existing Events behavior is not destructively moved by this blind patch because the current project file was not available here. If the app already has an Events route/nav, this patch leaves it intact.
